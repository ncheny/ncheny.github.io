Beep (2019)

3D game of a morphing alien
Beep is a Unity developed stealth/adventure game that our team has revised which was made in 2017 for a game design course.

User play as a morphing alien who crash landed on a foreign planet with hostile enemies and a hidden history.
  • TASKS Concept art, illustration, sound design, 3d modelling, researching
  • TOOLS Autodesk Maya, Unity3D, Clip Studio, Photoshop
  • TEAM Danielle Ragas, Josephine Ryu, Wan-teen Yang, Chris Wong
  • Spring 2019

Play the game Development Blog Beep Beta (2017)

Reworking & Planning

Beep (2019) is a revisit of a game our team made for our game design course in 2017 that we want to improve for our graduation project. Due to time constraint, we weren't able to implement a lot of our ideas in the initial Beep Beta (2017) which had many bugs, inconsistent environment, and a rushed narrative.

Since we have gotten the core game mechanics down, the major redesign initially was focused on the game environment and art style. Later we moved on to re-working the level design with some revisits to improving game mechanics.

During the initial stages, I looked at other games that we want to emulate visually and made quick environment concept sketches to explore the moods of different environment ideas. Many of these sketches references real life architecture, like the Lanyang Museum.

Level Design

I created concept sketches of level mechanics, puzzles, and level progression that could be implemented. It was at this stage that we decided on a changing environment, from pastoral landscapes to industrializing clear-cutted fields to tall buildings and machineries.

To make sure that user is experiencing not only story and time progression in the environment from morning to night, I made sure each area is dominated by a shade of colour palette to further reinforce a changing environment.

3D Modeling & Texturing

I was responsible for designing, modelling, and texturing various industrial 3D objects in the game. These include: doors, pillars, topiary, lattice, different types of walls. In addition, I also natural objects like leaf foliage and grass that are seen in the earlier parts of the game.
We first start by researching different styles, sketching it out, then creating it in Maya. Texturing is also done in Maya, then we would export it into Unity. After it is imported into Unity, a few more set up steps are needed, before it can be placed into the environment for interaction. Usually, different types of an environment object were created so my team can choose which they like best.

Level Building

We used a combination of Probuilder and Maya to build the scenes in our levels. To set up the scenes, I had to do a little research on how to incorporate Polybrush and Probuilder in combination with Unity native terrain engine. Researching and creating a fast pipeline allowed our group to quickly place in 3D assets into the environment for testing and world buidling.

I built multiple versions of level 2, helped set up 3D objects for other levels, and created the environment of level 4's entire map. From lighting, modeling, texturing, asset placement, and sound design, I worked closely with our puzzle designer, Josephine, to achieve the user experience we had envisioned.

UX & Sound

As the main sound designer, I collected sounds that can be used in the game, communicate with the coding team as to where to place them, set up the sound in Unity 3D, and edit the sound depending on what is needed. All the cutscenes' sound implementation was done through Unity as well.

Our game's main feature is to communicate the gameplay diegetically without relying on any text. Therefore, sound plays an important role of giving player feedback, since we have very little UI elements. I focused on incorporating sound seamlessly so user can have a smooth experience while playing the game.

Game Trailer & Thoughts

Midway through creating the game, we realized once again that we have a lot of ideas we wanted to do, but not enough time. Instead of 6 levels, the last two was condensed into one, but even then, we were falling a bit short on time. The last level turned out a bit buggy then we had anticipated it to be. Same can be said of level 2 and 4, which required a few patches to fix after the game was released online.

Regardless, we had succeeded in revamping our old Beep (2017) into one that is more polished, with a complete narrative, more cohesive game environment and sound design. Beep (2019) is a 3D adventurous game exploring an immersive gaming experience through non-verbal communication that is fun and satisfying for the user to play.

The game has received many positive gameplay reviews, Youtube playthroughs, and was featured in an PCGamer article. Game trailer was made by J.Ryu.
